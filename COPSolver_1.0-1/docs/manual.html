instalation guide for elementary OS

download COPSolver repository from https://github.com/tbfraga/COPSolver

or clone COPSolver repo using cmd $ clone https://github.com/tbfraga/COPSolver.git

go to download folder and unzip COPSolver_1.0-1.zip

go to COPSolver_1.0-1 using terminal

$ cd Downloads/COPSolver_1.0-1

configure system

$ ./config

install .deb file

$ sudo dpkg -i COPSolver_1.0-1.deb

$ if you want you can now run COPSolver using terminal

$ ./copsolver

Ready !!! You can now access COPSolver from the elementary OS Applications menu :)

using COPSolver on elementary OS

Once the solver has been installed, just click on the Applications menu and then on the COPSolver icon.

(1) Size the terminal window if you want and then press enter.

(2) Type 1, 2 or 3 to choose the problem definition method (as explained bellow) and then press enter.

Depending on the chosen method, COPSolver will work differently.

If you type 1:

COPSolver will come up with an optimal solution for the problem specified in the COPSolver/data.txt file.

If you type 2:

(3) Type 1, 2 or 3 to select one of the solver's predefined problems.

COPSolver will then come up with an optimal solution for the specified problem.

If you type 3:

(3) Enter an interger source value to generate a random problem and press enter.

(4) Enter an integer value to determine the number of products in the batch and press enter.

COPSolver will then come up with an optimal solution for the specified problem.

using data.txt for defining a benchmark

If you use standart installation, data.txt will be located at /Ä§ome/*user*/COPSolver/data.txt

where *user* is the user account.

you can change data.txt file as specified below:

first line: specify only one integer value for informing the number of products (N);

second line: specify N integer values for informing the production rate of each product;

third line: specify N integer values for informing the demand of each product;

fourth line: specify only one integer value for informing the outlets total free inventory;

fifth line: specify N integer values for informing the outlets free inventory for each product;

sixth line: specify only one integer value for informing the factory total free inventory;

seventh line: specify N integer values for informing the factory free inventory for each product;

eighth line: specify only one integer value for informing the multi-product batch processing time limit.

outputs

COPSolver will generate the following outputs:

output.txt: a file containing the last solver solution;

problem.txt: a file containing the last problem definition;

data.ldt: a LINGO data file with the last problem definition;

output.txt problem.txt files will be located at /home/*user*/COPSolver;

data.ldt file will be located at /home/*user*/COPSolver/LINGOSolver/MBPTM.

LINGO solver

Inside the /home/*user*/COPSolver/LINGOSolver/MBPTM folder there is a code in LINGO to solve the mathematical model developed for the muti-product batch processing time maximization problem.
